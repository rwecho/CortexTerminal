Architecture

三段式架构，强制中继，不走野路子

Mobile 发起会话，Gateway 做身份与路由治理，Worker 托管真实 AI CLI 进程。 整个系统围绕 Session continuity、Auditability、E2EE assumptions 构建。

Mobile Client

MAUI Hybrid + React + xterm.js，负责 session 发起、terminal UI 与移动端体验。

Gateway Relay

ASP.NET Core 10 + SignalR + PostgreSQL + Redis，负责 auth、session mapping、audit 与 binary relay。

Worker Node

Worker hosts AI CLI runtimes，负责 PTY lifecycle、session orchestration、ring buffer 与 reconnect snapshot。

Capabilities

不是 demo terminal，而是 product-grade terminal control plane

统一调度 Multi-runtime

一个 worker 可按 session 的 agent family 调度不同 CLI runtime，而不是把节点锁死成单一模型。

可审计会话链路

gateway 维护 session / worker / trace 视角的完整治理信息，便于 operator 与团队回溯。

断线重连友好

ring buffer + snapshot 设计确保移动端重连时不会轻易丢失上下文，终端状态可恢复。

保留原生 CLI 交互

PTY 层保留进度条、确认框、ANSI output 与真实交互，而不是阉割成“聊天框模拟器”。

Worker Onboarding

短 install token + 一条命令接入 Worker

在 mobile 里生成短期 install token 后，直接复制下面这一条命令到你的电脑执行。 脚本会自动下载 worker release、写默认配置、向 gateway 换取 registration key，并启动首个 worker。

One-liner
curl -fsSL 'https://gateway.ct.rwecho.top/install-worker.sh?token=iwk_xxxxxxxx' | bash

为什么是短 token？

它是一次性 install token，不是长期 access token。短期、一次性、服务端存状态，既更安全也更适合复制粘贴。

为什么不手填 runtime？

worker 会优先按显式配置读取；没有配置时，自动探测本机已安装的 Claude、Codex、Gemini、OpenCode CLI。
